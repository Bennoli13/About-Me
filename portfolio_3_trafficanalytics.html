Project Overview

A project focused on developing an alerting system to detect abnormal and error traffic patterns.

User Pain Points Addressed

  • Difficulty in detecting and responding to unusual traffic patterns in real-time.
  • Challenges in managing and setting traffic thresholds for different domains.
  • Inefficient notification process to domain owners regarding traffic anomalies.

Project Goals

  • To create a reliable system for monitoring and alerting on abnormal traffic behavior.
  • Implement domain-based threshold management using SQL databases.
  • Enhance the notification system for immediate alerts to domain owners.

Technical Details

  • Utilized Elasticsearch as the data source for traffic analysis.
  • Employed Python and Pandas for data analysis and traffic pattern identification.
  • Integrated a domain-based monitoring system where thresholds are managed via SQL database, with default thresholds for unconfigured domains.
  • Utilized Zabbix for scheduling checks, logging, and management ease.
  • The system includes a feature to automatically notify domain owners about traffic anomalies.
