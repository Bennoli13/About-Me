Project Overview

This project aimed to develop a centralized system for service monitoring and enhanced incident reporting using advanced technologies.

User Pain Points Addressed

  • Lack of centralized monitoring for quick identification of service disruptions.
  • Inefficient communication with customers during incidents, leading to dissatisfaction.
  • Delayed response times in addressing service issues.

Project Goals

  • To establish a centralized monitoring system using Zabbix v6.
  • Enhance customer communication during incidents using StatusPage.
  • Integrate a language model (VEXT) for clear, real-time incident reporting.

Technical Details

  • Set up a centralized monitoring system to cover all Mlytics services, including the Mlytics Portal, Origin Shield, Digital Experience Monitoring, and others.
  • Integrated StatusPage for better visibility of service uptime to customers.
  • Implemented VEXT for automated incident summary generation.
  • Developed a Telegram Bot integration with the Mlytics SOC to broadcast incident information.
