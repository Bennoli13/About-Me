Project information

  • Category: Web Application
  • Client: Internal Mlytics SOC Team

Project Overview

I initiated and led an internal project to develop a web-based tool for our SOC team. The tool's primary objective is to facilitate routine tasks like customer information retrieval, basic troubleshooting, and bulk configuration.

Objective

To streamline the SOC team's operational efficiency by providing a centralized, web-based platform for routine tasks and information access.

User Pain Points Addressed

  • Inefficient access to customer data and troubleshooting tools.
  • Cumbersome bulk configuration processes.
  • Lack of a unified platform for routine SOC activities.

Project Goals

  • Develop a user-friendly, web-based tool to centralize key SOC functionalities.
  • Implement Agile methodologies for continuous improvement and deployment.
  • Ensure high availability and security of the application.

Technical Details

  • As the sole developer, I followed Agile methodologies, prioritizing features and fixes in each sprint.
  • The application was built using the Python Flask framework, chosen for its efficiency in handling multiple API interactions.
  • Deployed in Docker containers for easy management, running on Google Cloud VM.
  • Monitoring of server availability was set up using Zabbix.
  • Integration with various databases like SQL and MongoDB as the application scaled.
  • Utilized Google OAuth for secure access, with users logging in via company email.

Summary

This page showcases my comprehensive skills in project management, deployment, and development, particularly in creating a Python-based web app tailored to SOC team needs.
